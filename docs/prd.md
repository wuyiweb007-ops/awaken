# 工字日程纸 · H5 小产品 PRD（思路版）

版本：v1.2  
范围：三页 —— 「今天」「历史」「我的」，移动端 H5

---

## 1. 产品概念与设计原则

**产品一句话**  
在手机上复刻《认知觉醒》的“工字型日程纸”，帮助用户通过「规划 → 拆解 → 落地 → 省察」这四步，把一天过得更清醒，而不是更忙。

**四步重新定义**

1. **规划（目标层）**：用户心里对“今天想成为什么样的一天”的大致图景。这里不单独做一页，而是散落在待办和计划描述里。
2. **计划（拆解任务）**：在「计划完成」区用时间段 + 描述的方式，给目标拆分出若干时间块。
3. **完成（实际情况）**：在「实际完成」区记录真实发生的时间段 + 事情，不要求与计划一一对应，允许计划外事项。
4. **省察（反思调整）**：在「省察感悟」区用简单语言写下今天的观察与调整点。

> 重点：**计划和完成是两个平行视角** —— 计划是“我打算怎么用时间”，完成是“我实际上怎么用时间”，只提供从计划复制到完成的加速操作，不做一一绑定和强制校验。[web:187][web:190]

**整体设计原则**

- 一屏完成：在常见手机尺寸下，「今天」页核心内容尽量在一屏内可见，只允许小范围滚动。
- 纸质体验：整体 UI 为“纸质卡片风”，避免过多图表和组件感，尽量像一本小册子。
- 轻操作：所有高频操作（写今天、看昨天、设提醒）都控制在 1–2 步内完成。
- 认知优先而非 gamification：激励只做轻度反馈（完成小动画、细微动效），不追求重度打卡和连胜压力。[web:186][web:190]

## 2. 页面与导航

底部 Tab 三个入口：

1. 「今天」：工字日程纸首页（默认页）。
2. 「历史」：按日期翻页查看历史记录，纸质翻页效果。
3. 「我的」：解释来源与用法 + 提醒设置等。

Tab 图标和文字都采用简单、易理解的符号和标签，避免抽象图标带来的认知负担。[web:186]

## 3. 「今天」页 —— 工字日程纸

### 3.1 布局与风格

- 顶部栏：
  - 左：标题「工字日程纸」。
  - 右：当前日期（`YYYY-MM-DD 周X`），不显示时间与其他状态，减少占位。
- 主体：标准“工字”结构（竖屏）：
  - 顶部整行：① 待办事项（规划层，横向铺满）。
  - 中间左列：② 计划完成（时间块拆解）。
  - 中间右列：③ 实际完成（真实发生的时间块）。
  - 底部整行：④ 省察感悟（反思）。
- 底部栏（紧凑）：左侧为 `计划 X.X 小时 · 实际 Y.Y 小时`，右侧为「✓ 记下这一天」。

**UI 风格**

- 背景为浅米色/浅黄纸张纹理，四区像印在一张卡纸上。
- 分割线、边框为浅灰/浅棕细线，模拟纸上的线条。
- 字体统一使用易读的系统无衬线中文 + 轻微字间距。
- 激励与动效是“小动作”，例如：
  - 完成某个计划块时，在实际区生成对应条目时有轻微缩放 / 勾选动画，而非强刺激。

### 3.2 功能与交互

#### ① 待办事项（顶部）

- 功能：把今天想做的事全部列出来，作为规划的“素材池”。
- 形式：多行文本，每行包括：
  - 可选优先级标签（A/B/C/空，循环）。
  - 文本输入框（自由描述）。
- 交互：
  - 初始生成若干空行。
  - 点击“＋ 添加一行”增加。
  - 不做完成勾选，完成情况由「实际完成」体现。

#### ② 计划完成（中左）

- 功能：将重要待办拆解为带时间段的计划块。
- 每一条计划块：
  - 时间段输入：`HH:MM-HH:MM`。
  - 内容描述：短句，如「首页开发」「阅读《认知觉醒》」。
- 交互：
  - 初始生成若干空块。
  - 用户可以从待办区点击某项「发送到计划」，自动填充描述，时间仍需手动填。
  - 修改时间或新增/删除块时，自动更新「计划总小时数」。

#### ③ 实际完成（中右）

- 功能：记录真实发生的时间段和事项，允许计划外内容。
- 每一条完成块：
  - 时间段：`HH:MM-HH:MM`（实际起止时间）。
  - 内容描述：默认基于来源计划生成，但允许编辑，如：
    - 计划：`10:30-12:30 首页开发`
    - 完成（从计划生成）：`10:30-12:30 完成首页开发`
  - 来源标记（可选小图标）：标记是否来自某个计划，方便用户感知“计划内/计划外”。
- 交互要点：
  1. **从计划带入**
     - 在计划块旁提供“小对勾”或「移到完成」操作。
     - 点击后，在完成区生成一个新条目：
       - 时间段默认使用计划时间；可立即编辑为真实时间。
       - 描述默认在计划描述前加“完成/处理”等词，也可以只复制原文。
     - 动效：生成时有轻微勾选 + 完成条目淡入的动画（轻激励）。
  2. **自由新增完成块**
     - 在完成区可以直接新增一条空记录，用于记录计划外发生的事情（如突发会议）。
  3. **非一一对应**
     - 同一个计划块可以多次生成不同完成块（如果今天分段完成）。
     - 完成区可以存在完全没有计划来源的记录。

#### ④ 省察感悟（底部）

- 功能：用几句话总结今天的观察和调整方向。
- 形式：多行文本区域，预置简短提示（如“今天最满意的是…/今天时间主要花在…/明天想调整的是…”）。
- 不做结构化字段，仅作为自由文本，方便后续历史中直接阅览。

#### 总体统计与保存

- 统计：
  - 计划小时数：所有合法计划时间段累计；只用来提示，不作为硬指标。
  - 实际小时数：所有完成时间段累计。
- 保存逻辑：
  - 每次用户修改（计划/完成/省察）时自动缓存在本地；「✓ 记下这一天」则作为“心理上收工”的动作，附带轻微动效或提示文案。

## 4. 「历史」页 —— 纸质翻页式回看

**目标**  
让用户像翻一本纸质手帐一样，顺畅地向前/向后翻看每天的工字日程纸，看到的是“真实的自己”，而不是流水账列表。

### 4.1 布局与风格

- 进入「历史」后，默认展示“上一日”的完整工字纸（非列表）。
- 屏幕中央是一整张“卡纸”（布局与「今天」页类似，但为只读）。
- 屏幕右侧/左侧有简洁的日期时间轴与翻页提示。

### 4.2 交互设计

- 左右滑动翻页：
  - 向左滑：查看更早的一天。
  - 向右滑：返回更近的一天（直至今天）。
- 滚动翻页：
  - 纵向轻微滑动只在当前卡纸内部滚动（当内容超出一屏时）。
  - 侧边可显示一条简化时间轴（竖线 + 年月标记），用户拖动小滑块可以快速跳转某个时间段。
- 纸质翻页特效：
  - 翻页时模拟纸张翻动效果：当前卡纸略微卷起、淡出，新卡纸从另一侧滑入。
  - 动效保持轻量，避免炫技影响性能和体验。
- 日期显示：
  - 顶部显示当前查看日期，格式 `YYYY-MM-DD 周X`。
  - 若该日无记录，显示“这一天没有写工字纸”的空态，可以快速跳转到最近有记录的一天或回到「今天」。

## 5. 「我的」页 —— 来源说明与提醒中枢

**目标**  
一是告诉用户：这个 App 的理论来源、自用方法和作用是什么；二是成为“提醒中心”，真正帮用户养成“写今天 / 看昨天”的节奏。

### 5.1 布局概览

- 顶部文案：「关于工字日程纸」。
- 接下来两个模块：
  1. 理念与用法说明。
  2. 提醒与基础设置（高优先级）。

### 5.2 理念与用法说明（结合《认知觉醒》）

- 短文说明三个问题：
  1. 这个 App 的灵感源自哪套方法（工字型日程纸 + 早冥读写跑等认知觉醒理念，避免复述书中原文）。
  2. 它想解决什么问题：不是让你填满一天，而是让你看清自己怎么用一天。
  3. 怎么用：
     - 早/前一晚在「今天」写待办和计划。
     - 白天有进展就及时在「实际完成」里记一下。
     - 晚上回来看差别，并在省察里写几句。
- 文案语气偏“陪练”，不做说教。

### 5.3 提醒与设置（重点）

**闹钟提醒（必须实现）**

- 早间提醒：
  - 开关 + 时间选择（如 07:30）。
  - 文案示例：“提醒我写今天的工字纸”。
- 晚间提醒：
  - 开关 + 时间选择（如 21:30）。
  - 文案示例：“提醒我回来看今天的完成和省察”。
- 实现方式：
  - 如果在 App WebView 内，可以调用宿主的本地通知能力。
  - 如果是纯 H5，可采用浏览器通知 / 系统层提醒的折中方案，视技术栈决定。
- UX 要点：
  - 默认关闭，由用户自选开启，避免“通知骚扰”。[web:187][web:190]
  - 提醒文案与 App 语气一致，简短温和。

**其他轻设置**

- 默认打开页面：今天 / 历史（二选一）。
- 主题风格：浅色纸张 / 深色纸张 / 跟随系统。
- 数据管理（选做）：一键导出或清空本机数据，带明确风险提示。

## 6. 总体体验与风格说明

### 6.1 交互思路

- 「今天」页：**写**与**看**都控制在一屏，不跳转子页，所有编辑在原位完成。
- 「历史」页：像翻纸质手帐一样的翻页体验，用户只需左右滑动即可在时间上前后穿梭。
- 「我的」页：多为开关、单选，不设计复杂表单；闹钟提醒是唯一刚需设置，其余都可选。

### 6.2 视觉与动效

- 主色：温暖纸张色（米黄、浅棕），强调“手帐 / 纸质感”。
- 分隔线与边框：细而浅，避免信息过于“工具化”，让用户心理上更接近日记本而非工作系统。
- 字体大小：保证在手机上 1 屏可读，且控件触达区域符合手机 UX 最佳实践（最低 44x44dp）。[web:174][web:190]
- 动效：
  - 完成计划 → 生成完成条目时，有小勾选和轻微放大/淡入。
  - 历史翻页是纸张翻动动画，但要短（200–300ms），以流畅为主。
  - 其余交互动效保持极简，避免“炫酷但分散注意力”。

## 7. 对开发的抽象要求

- 「今天」页面的数据模型要允许：
  - 计划和完成互相独立，但完成条目可以标记来源于哪个计划。
  - 完成条目可以与计划条目数量不同（多于或少于）。
- 本地数据持久化：
  - 每天一条记录，以日期为主键存储四个区块内容和统计字段。
- 通知/闹钟：
  - 需要一套统一的提醒 API 封装，供“早间提醒”“晚间提醒”配置调用。
- 历史翻页：
  - 以“按日分页”的方式渲染，而非长列表；翻页动效可在视图层实现。
